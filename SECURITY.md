# Security Policy

## Supported Versions
Security fixes (if any) are provided for the **latest release** only.

## Reporting a Vulnerability
If you believe you have found a security vulnerability in OCRtoODT:

1. **Do not** open a public GitHub issue if the report includes exploit details.
2. Instead, contact the maintainer via GitHub:
   - Open a new issue with the title **"Security report (private details available)"** and include only high-level information, **or**
   - If you prefer, contact the maintainer through GitHub profile contact options.

Please include:
- OCRtoODT version (e.g. `vX.Y.Z`)
- OS and distribution
- Steps to reproduce / proof-of-concept (if safe)
- Impact assessment (what can be achieved)

## Response Expectations
- You will receive an initial response as soon as possible.
- If the report is valid, a fix will be prepared and released.
- After a fix is released, public disclosure can be coordinated in the related issue/release notes.
